Effort level for MIP starts in CPLEX V12.7.0

Announces a new level of effort for MIP starts as a new feature of CPLEX V12.7.0.

CPLEX V12.7.0 offers a new value (CPX_MIPSTART_NOCHECK) to specify the level of effort that CPLEX should expend when it applies a MIP start (also known as a warm start or advanced start). You pass this new value to the routine or method that declares the MIP start to CPLEX. The new level of effort specifies to CPLEX that you want to inject a complete solution as the incumbent without CPLEX performing the usual checks. CPLEX checks whether the MIP start is a complete solution; if the MIP start is not a complete solution, CPLEX rejects it. If it is a complete solution, CPLEX injects it as the incumbent and proceeds with no further checking. At this level, CPLEX performs no other checks. Consequently, if the solution defined by the MIP start is infeasible, behavior is undefined.

  • In the Callable Library (C API), use this new value in the routine CPXXaddmipstarts
  • In the C++ API, use this new value in IloCplex::addMIPStart
  • In the Java API, use this new value in IloCplex.addMIPStart
  • In the .NET API, use this new value in Cplex.AddMIPStart
  • In the Python API, use this new value in cplex.MIP_starts.add
  • In the Interactive Optimizer, use this new value as an option in the file that you read to establish the MIP start.

Tip: The MIP start should be a complete solution when you declare this level of effort.

You can also use this new value for the effort level in other routines of the Callable Library (C API) to access or modify MIP starts, such routines as CPXXgetmipstarts or CPXXchgmipstarts. Likewise, you can use this new value for effort level in methods of the C++, Java, .NET, Python APIs to access or modify MIP starts. In other words, this new effort level is not limited to creation of a MIP start.

For more about MIP starts, see the topic Starting from a solution: MIP starts in the CPLEX User's Manual, especially the subtopic MIP starts and effort level.
