New Java features in CPLEX V12.6.3

Announces new features of CPLEX V12.6.3 in the Java API.

These notes highlight new features in the Java API of interest to users of CPLEX V12.6.3.

New support for Community Edition in the Java API

To enable users of the Community Edition of IBM ILOG CPLEX Optimization Studio to verify that the size of their model does not exceed limits of the Community Edition, CPLEX V12.6.3 now offers new methods. See the following announcements about new counting methods for this purpose.

New method to count indicator constraints in Java API

CPLEX V12.6.3 offers a new method IloCplex.getNindicators in the Java API. This method returns the number of indicator constraints in a model. It counts explicit indicator constraints in the model as well as constraints added implicitly when CPLEX expands a logical constraint into multiple indicator constraints automatically. This counting method can be useful, for example, to test whether the size of a model conforms to limits in the Community Edition of IBM ILOG CPLEX Optimization Studio.

New method to count lazy constraints in Java API

CPLEX V12.6.3 offers a new method IloCplex.getNLCs in the Java API. This method returns the number of lazy constraints in a model. This counting method can be useful, for example, to test whether the size of a model conforms to limits in the Community Edition of IBM ILOG CPLEX Optimization Studio.

New method to count user cuts in Java API

CPLEX V12.6.3 offers a new method IloCplex.getNUCs in the Java API. This method returns the number of user cuts in a model. This counting method can be useful, for example, to test whether the size of a model conforms to limits in the Community Edition of IBM ILOG CPLEX Optimization Studio.
