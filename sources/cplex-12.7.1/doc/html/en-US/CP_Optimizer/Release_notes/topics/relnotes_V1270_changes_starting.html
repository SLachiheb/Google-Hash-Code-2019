New feature: Starting points in the CPO File Format

Starting points can now be represented in the CPO File Format.

CP Optimizer warm starts (setStartingPoint) can now be represented in the .cpo file format in a startingPoint section. Including starting points in the file format makes it possible to persistently store starting points in a portable manner and communicate starting points with other users or IBM® support.

The following is an example of how to specify a starting point:

  x = intVar(1..10);
  y = intVar(1..10);
  x + y < 12;
  itv = intervalVar(optional, end=0..100, size=1..5);
  startOf(itv, 10) == x;
  endOf(itv, 1) == y;
  startingPoint {
    x = intVar(3);
    itv = intervalVar(present, size=4, start=7);
  }

Additionally, CP Optimizer's warning message mechanism was updated to include additional information on starting points, particularly when the starting point contains inconsistent information (such as values that are not possible for particular variables due to domain restrictions).
